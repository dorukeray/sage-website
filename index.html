Introducing Sage

Sage is a query-based data exchange protocol for APIs.

Sage aims to simplify the communication for data interactions between different layers of software, and designed especially for APIs.

Sage is simple data exchange.

Get Started Learn More

What is Sage?

Sage is an open protocol for designing and building better APIs, simply a specification about how to query/response for data and a runtime capable of executing those queries with your existing business logic and data.

Sage provides a simple & lightweight but also efficient, expressive, productive and intuitive way for describing the data in your API, giving clients the power to interact with your API exactly how they need to—performing data retrieval or actions—makes evolving and documenting your APIs easier.

We created Sage for catalyzing the Web 3.0 revolution.

Simplify data exchange.

Data exchange should be simple, lightweight and efficient. That’s what Sage is all about. Sage helps you building more powerful open data APIs.

1. Define your data schema
Describe your data as an entity graph, then write the code which maps your data schema to your existing business logic.
2. Want only what you need
Query your Sage API and get exactly what you want. Apps using Sage are performant and stable because they get predictable data.
3. Consume your data
Use your data wisely and develop your apps more productively than you thought possible, with any frontend technology.

Values

What you can expect?

Sage is much more than a "yet another approach for APIs", it is the way for introducing simplicity to the data exchange.

Query-based

With Sage, you query your data by declaring what you want; then get only that.

Any data consumption and creation process imaginable can be implemented with Sage.

Product-centric

Sage is primarily designed to solve data interaction problems in product engineering.

Sage give apps the ability to move fast without breaking things.

With your existing business logic and data

Sage creates a uniform API across your entire application by leveraging your existing data and business logic with a Sage runtime.

API evolution, not revolution.

Change your data schema without impacting existing queries and documentation.

With Sage, your schema is your contract — so you always get correct documentation.

Environment Agnostic

Sage never dictates the use of any programming language, backend, storage technique or even a query language like GraphQL.

Instead, Sage focuses on concepts and patterns that are achievable no matter how you're building a service.

Graph-like

Sage achieves harmony with the structure of today's modern data-driven apps.

To do this, Sage guides you through the creation, consumption and evolution of your data as a universal entity graph.
